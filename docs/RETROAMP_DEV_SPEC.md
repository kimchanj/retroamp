# 🎧 포트폴리오 2 --- RetroAmp (Winamp 스타일 데스크톱 오디오 플레이어) v1.0

> 목적: Winamp 감성의 데스크톱 오디오 플레이어를 구조 중심 설계로
> 구현하고,\
> 스프린트 단위 개발 + Git 이력 관리 기반으로 프로젝트를 확장한다.

------------------------------------------------------------------------

## 0. 프로젝트 개요 (Overview)

-   **프로젝트명(상품형 이름)**: RetroAmp
-   **한 줄 설명(가치 제안)**: "Winamp 감성 UI + 구조 중심 설계로 구현한
    데스크톱 오디오 플레이어"
-   **개발 목적**:
    -   단순 플레이어 구현이 아닌 **도메인 모델링 + 아키텍처 분리 +
        스프린트 운영 능력**을 보여주는 포트폴리오
    -   Electron + React + TypeScript 기반 데스크톱 앱 구조 이해
    -   기능 확장에 대비한 설계 중심 개발

------------------------------------------------------------------------

# 1. 요구사항 정의 (Requirements)

## 1-1. 사용자 요구사항 (User Stories)

-   US01. 사용자는 여러 개의 MP3 파일을 한 번에 선택할 수 있어야 한다.
-   US02. 선택한 파일은 플레이리스트 목록에 표시되어야 한다.
-   US03. 현재 재생 중인 트랙을 시각적으로 확인할 수 있어야 한다.
-   US04. 한 곡이 끝나면 자동으로 다음 곡이 재생되어야 한다.
-   US05. 이전/다음 버튼으로 곡 이동이 가능해야 한다.
-   US06. 재생 위치를 슬라이더로 조절할 수 있어야 한다.
-   US07. 볼륨을 조절할 수 있어야 한다.

------------------------------------------------------------------------

  ----- ---------------------------- ---------- ------

  F11   반복                         V2         ⬜

------------------------------------------------------------------------

## 1-3. 비기능 요구사항

-   NFR01. UI와 로직은 분리되어야 한다.
-   NFR02. 플레이리스트 로직은 UI에서 직접 제어하지 않는다.
-   NFR03. 기능 확장이 가능하도록 구조를 유지한다.
-   NFR04. 빌드/배포 후에도 동작 일관성을 유지한다.

# 2. 설계 (Design)

## 2-1. 아키텍처 구조

Electron Main\
→ Preload (IPC Bridge)\
→ Renderer (React UI)\
→ Domain (Playlist Model)\
→ Service (Audio Controller)

------------------------------------------------------------------------

## 2-2. 폴더 구조

src/ ├─ models/ │ └─ playlist.ts ├─ services/ │ └─ audioService.ts ├─
components/ │ ├─ PlayerControls.tsx │ ├─ DisplayPanel.tsx │ ├─
Slider.tsx │ └─ PlaylistPanel.tsx └─ App.tsx

------------------------------------------------------------------------

## 2-3. 도메인 모델 설계

### Track

-   id
-   path
-   title
-   duration

### Playlist 책임

-   add()
-   remove()
-   next()
-   prev()
-   getCurrent()
-   shuffle()
-   repeat()

설계 원칙: - UI는 직접 track index를 변경하지 않는다. - Playlist가 재생
순서를 책임진다.
